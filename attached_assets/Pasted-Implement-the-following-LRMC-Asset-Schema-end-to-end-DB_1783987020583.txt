Implement the following LRMC Asset Schema end‑to‑end:
DB schema, OpenAPI spec, API routes, web pages, admin workflows,
module integration, and metadata support. This schema is the
institutional model for LRMC.

3. UNIVERSAL ASSET FIELDS
--------------------------------
id                integer PK
owner_id          integer FK → users.id
category          text
type              text
title             text
description       text
status            text
metadata          jsonb
created_at        timestamp
updated_at        timestamp

Status values:
  - pending_review
  - approved
  - rejected
  - archived
  - active
  - inactive

5. ASSET RELATIONSHIPS
--------------------------------
Assets link to:
  - Members (owner, manager, admin reviewer)
  - Attachments (photos, documents, proofs, contracts)
  - Modules (Property, Ususu, Marketplace, Digital, Employment, Travel, Events)
  - Revenue (payment plans, subscriptions, commissions)
  - Workflows (approval, review, assignment, payment)

6. ASSET LIFECYCLE
--------------------------------
1. Created
2. Pending Review
3. Reviewed
4. Approved or Rejected
5. Activated
6. Assigned to Module
7. Linked to Revenue
8. Updated
9. Archived

7. ASSET APPROVAL MODEL
--------------------------------
Member actions:
  - Create asset
  - Upload attachments
  - Edit pending assets
  - Delete pending assets

Admin actions:
  - Approve
  - Reject
  - Delete
  - Assign to module
  - Link to revenue
  - Link to workflows

8. ASSET API CONTRACT (OpenAPI)
--------------------------------
GET    /assets
POST   /assets
GET    /assets/{id}
PATCH  /assets/{id}
DELETE /assets/{id}

POST   /assets/{id}/approve
POST   /assets/{id}/reject
POST   /assets/{id}/assign
POST   /assets/{id}/link-revenue

9. ASSET WEB PAGES
--------------------------------
/assets:
  - Card grid
  - Filters (category, type, status)
  - "My Assets" toggle
  - Create dialog
  - Admin approval actions

/assets/{id}:
  - Full asset detail
  - Metadata viewer
  - Attachments viewer
  - Workflow history
  - Revenue links
  - Module links

10. MODULE INTEGRATION
--------------------------------
Modules must be able to:
  - Read assets
  - Filter assets
  - Assign assets
  - Link assets
  - Display assets
  - Use assets in workflows
